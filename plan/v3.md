# V3 Implementation Steps

Status: complete.

1. Define the V3 scope as a browser-local Chinese Pinyin module foundation.
2. Use the current built-in sample dictionary as the V3 data source.
3. Keep licensed large-corpus dictionary import as a future version step.
4. Define the dictionary data format:
   - syllable key
   - candidate text
   - frequency score
   - phrase length
   - optional metadata
5. Convert the built-in dictionary to structured entries.
6. Add multi-syllable phrase candidates.
7. Add multi-syllable segmentation for continuous Pinyin input.
8. Rank candidates by frequency and phrase match quality.
9. Keep candidate pagination controls for larger candidate sets.
10. Add fuzzy Pinyin setting:
    - `z` / `zh`
    - `c` / `ch`
    - `s` / `sh`
    - `n` / `l`
11. Add Chinese punctuation handling in Pinyin mode.
12. Preserve plain-text commit behavior for unknown Pinyin buffers.
13. Defer local phrase learning until the dictionary source is larger.
14. Defer learned-data clearing until phrase learning exists.
15. Defer custom phrase import/export until there is a custom phrase format.
16. Add tests for segmentation and ranking.
17. Add tests for fuzzy Pinyin options.
18. Add browser tests for Pinyin settings, pagination, fuzzy input, and punctuation.
19. Audit bundle size after the dictionary changes.
20. Audit browser behavior with the full release check.
21. Update README with Pinyin limitations and privacy behavior.
22. Run full release checks.
