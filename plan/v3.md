# V3 Implementation Steps

1. Define the target scope for a full Chinese Pinyin module.
2. Choose a dictionary and phrase data source with a compatible license.
3. Define the dictionary data format:
   - syllable key
   - candidate text
   - frequency score
   - phrase length
   - optional metadata
4. Add a dictionary build/import script.
5. Keep generated dictionary artifacts separate from handwritten source code.
6. Add lazy loading for large Pinyin dictionary data.
7. Add multi-syllable segmentation for continuous Pinyin input.
8. Rank candidates by frequency and phrase match quality.
9. Add candidate pagination controls for large candidate sets.
10. Add fuzzy Pinyin settings:
    - `z` / `zh`
    - `c` / `ch`
    - `s` / `sh`
    - `n` / `l`
11. Add punctuation handling for Chinese and English punctuation modes.
12. Add mixed English and Chinese typing behavior.
13. Add optional user phrase learning stored locally.
14. Add controls to clear learned local Pinyin data.
15. Add import/export for custom Pinyin phrases if needed.
16. Add tests for segmentation and ranking.
17. Add tests for fuzzy Pinyin options.
18. Add browser tests for long Pinyin input and pagination.
19. Audit dictionary load performance.
20. Audit typing latency during composition.
21. Update README with Pinyin limitations and privacy behavior.
22. Run full release checks.
