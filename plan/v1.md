# V1 Implementation Steps

1. Define the 0.2.0 V1 scope as shared optional appearance styling for the whole input panel.
2. Keep the current flat virtual keyboard and mouse as the default `simple` appearance.
3. Add one shared input panel appearance setting for both keyboard and mouse.
4. Persist the selected input panel appearance in local storage.
5. Support appearance options:
   - simple
   - realistic
6. Add one shared skin setting for the whole input panel.
7. Persist the selected input panel skin in local storage.
8. Support at least three skins:
   - classic light
   - dark mechanical
   - natural wood
   - high-contrast
9. Define skin as CSS-only texture, pattern, color, border, shadow, and pressed-state styling.
10. Do not use image assets, canvas, SVG illustration, or 3D for V1 realism.
11. Avoid storing user-created skins or custom assets.
12. Update the Panel menu to expose shared appearance and skin controls without crowding the menu bar.
13. Hide or disable the skin control when `simple` appearance is active if it would be misleading.
14. Make realistic keyboard keys look more physical with depth, shadows, rounded keycaps, and pressed states.
15. Make realistic mouse buttons look more physical with a body shape, separated buttons, depth, and click feedback.
16. Apply the same appearance and skin to keyboard and mouse; do not allow separate keyboard and mouse styling.
17. Ensure physical key highlighting still works in every appearance and skin.
18. Ensure virtual key click insertion still works in every appearance and skin.
19. Ensure mouse click feedback still works in every appearance and skin.
20. Keep the panel bottom-anchored and prevent appearance or skin changes from changing layout height unexpectedly.
21. Keep realistic panel skins independent from the app theme.
22. Let the app theme affect the panel only when the panel appearance is `simple`.
23. Ensure controls remain accessible with labels, keyboard focus, and visible focus states.
24. Add unit tests for appearance and skin setting parsing or helpers if helpers are introduced.
25. Add browser smoke coverage for switching appearance and skin.
26. Add visual screenshot coverage for the realistic keyboard and mouse appearance.
27. Update README manual checks for the new panel appearance controls.
28. Complete V1 locally only; do not deploy V1 until user review.
29. Keep V1, V2, and V3 available for a combined 0.2.0 release later.
30. Run `npm run test`.
31. Run `npm run test:browser`.
32. Run `npm run release:check`.
33. Run `GITHUB_PAGES_BASE=/freetyping/ npm run build`.
