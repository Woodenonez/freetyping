# V3 Implementation Steps

1. Define the 0.2.0 V3 scope as refactoring input behavior so the app adapts better to multiple languages and layouts.
2. Replace the current input-mode model with only two top-level Input Mode options:
   - System Input
   - Overlay Input
3. Keep `System Input` behavior unchanged:
   - browser and operating system input control actual text entry
   - selected panel layout is only visual/reference unless virtual keys are clicked
4. Define `Overlay Input` as app-controlled input where actual behavior depends on the selected panel layout.
5. Move language-specific input behavior out of the top-level Input Mode menu and into layout-driven overlay handlers.
6. Under the Panel menu, support these layout families:
   - QWERTY
   - Pinyin (CN)
   - country-specific Nordic layouts, merged when their key mappings are identical
7. Remove the `Combined Nordic` layout from the product behavior.
8. Replace `Nordic Direct` input mode with layout-driven Nordic overlay handlers.
9. Define Nordic layouts by actual key mapping, not by country count.
10. Merge countries into one visible layout option when their implemented key maps are identical.
11. Use combined country codes in the label for merged layouts, for example `Nordic (SE/FI)` if Sweden and Finland share the same implemented mapping.
12. Keep countries separate when their implemented key maps differ, for example `Nordic (NO)` or `Nordic (DK)` if their special-letter or symbol placement differs.
13. Do not ship duplicate Nordic layout options that behave identically.
14. Document which countries each Nordic layout option covers in code comments or layout metadata.
15. Ensure each Nordic layout maps physical key codes according to that country or merged-country layout where browser `event.code` supports it.
16. Ensure each Nordic virtual keyboard displays the exact characters inserted by its overlay handler.
17. Keep QWERTY as the default layout.
18. Make `Overlay Input` with QWERTY behave like direct QWERTY text entry.
19. Make `Overlay Input` with `Pinyin (CN)` use the existing Chinese Pinyin composition and candidate logic.
20. Make `Overlay Input` with a Nordic layout use that selected Nordic mapping for physical and virtual input.
21. When the user selects `Pinyin (CN)`, automatically switch Input Mode to `Overlay Input`.
22. When the user selects any Nordic layout, automatically switch Input Mode to `Overlay Input`.
23. When the user selects `QWERTY`, keep the current top-level Input Mode if it is valid.
24. Do not disable layouts based on Input Mode; layout remains independently selectable from the Panel menu.
25. Remove the old top-level input mode options:
   - English Direct
   - Nordic Direct
   - Chinese Pinyin
26. Keep Pinyin settings available only when `Overlay Input` and `Pinyin (CN)` are active.
27. Keep Pinyin dictionary loading lazy and browser-local.
28. Update internal names and types so `inputMode` means only `system` or `overlay`.
29. Add a layout-overlay controller or equivalent abstraction that chooses behavior from the selected layout.
30. Keep virtual key click insertion consistent with the selected layout and overlay behavior.
31. Preserve physical key highlighting for all layouts where `event.code` maps cleanly.
32. Preserve V1 appearance and skin behavior for all layouts.
33. Update local storage keys carefully:
   - migrate old `nordic` combined layout to `qwerty` so users explicitly choose a Nordic variant
   - migrate old `nordic-direct` input mode to `overlay`
   - migrate old `zh-pinyin` input mode to `overlay` plus `pinyin-cn` layout
   - migrate old `en-direct` input mode to `overlay` plus `qwerty` layout
34. Add unit tests for input-mode migration and layout selection helpers.
35. Add unit tests for Nordic layout data, including assertions that identical mappings are merged and differing mappings stay separate.
36. Add browser smoke coverage for:
   - System Input unchanged
   - Overlay Input with QWERTY
   - Overlay Input with Pinyin (CN)
   - Overlay Input with each visible Nordic layout option
   - migration from old V2 local storage values
37. Add screenshot coverage for each visible Nordic layout option.
38. Update README manual checks for the new System/Overlay model.
39. Complete V3 locally only; do not deploy V3 until user review.
40. Keep V1, V2, and V3 available for a combined 0.2.0 release later.
41. Run `npm run test`.
42. Run `npm run test:browser`.
43. Run `npm run release:check`.
44. Run `GITHUB_PAGES_BASE=/freetyping/ npm run build`.
