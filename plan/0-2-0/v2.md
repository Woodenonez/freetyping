# V2 Implementation Steps

1. Define the 0.2.0 V2 scope as adding a combined Nordic keyboard layout and Nordic direct input mode.
2. Keep the current QWERTY keyboard as the default layout.
3. Add one keyboard layout setting for the virtual keyboard.
4. Persist the selected keyboard layout in local storage.
5. Support virtual keyboard layout options:
   - QWERTY
   - Combined Nordic
6. Define Combined Nordic as a practical FreeTyping layout, not a country-perfect physical keyboard.
7. Do not add country-specific layout variants in V2.
8. Include Nordic character keys directly on the visible virtual keyboard:
   - `Å`
   - `Ä`
   - `Ö`
   - `Æ`
   - `Ø`
9. Add Nordic punctuation and symbol keys where they are useful for the combined layout.
10. Keep key labels readable and stable inside the existing virtual keyboard dimensions.
11. Add a `Nordic Direct` input mode.
12. Make `Nordic Direct` insert exactly the clicked virtual key character.
13. Keep physical typing in `System Input` controlled by the user's OS/browser keyboard.
14. Selecting Combined Nordic layout should switch input mode to `Nordic Direct` by default.
15. Selecting `English Direct` should switch the virtual keyboard layout back to QWERTY.
16. Selecting `Chinese Pinyin` should switch the virtual keyboard layout back to QWERTY.
17. Selecting `System Input` should not force a layout change; users can view either virtual layout while system input is active.
18. Disable non-Nordic web input modes while Combined Nordic layout is active.
19. Ensure Chinese Pinyin mode remains QWERTY-letter based and does not capture Nordic character keys as Pinyin letters.
20. Update virtual keyboard data structures so layouts can be selected without duplicating rendering logic.
21. Ensure physical key highlighting still works where browser `event.code` maps cleanly to the selected layout.
22. Ensure virtual key click insertion inserts the character shown on the key.
23. Add layout controls under the Panel menu:
    - Keyboard checkbox
    - Mouse checkbox
    - divider
    - layout options
24. Disable layout controls when the Keyboard panel is unchecked.
25. Ensure layout selection works together with V1 panel appearance and skin settings.
26. Add unit tests for Nordic layout data and key normalization if needed.
27. Add browser smoke coverage for selecting Combined Nordic layout and inserting Nordic characters from virtual keys.
28. Add browser smoke coverage for input mode and layout auto-switching rules.
29. Add visual screenshot coverage for the Nordic virtual keyboard.
30. Update README manual checks for Nordic layout behavior.
31. Complete V2 locally only; do not deploy V2 until user review.
32. Keep V1, V2, and V3 available for a combined 0.2.0 release later.
33. Run `npm run test`.
34. Run `npm run test:browser`.
35. Run `npm run release:check`.
36. Run `GITHUB_PAGES_BASE=/freetyping/ npm run build`.
