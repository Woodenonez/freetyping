# V1.1 Implementation Steps

1. Create `src/input/modes/chinesePinyin/`.
2. Add minimal dictionary data:
   - `ni`
   - `hao`
   - `nihao`
   - `zhong`
   - `guo`
   - `zhongguo`
3. Add candidate lookup helper.
4. Add Pinyin mode state:
   - active buffer
   - candidate list
   - selected candidate index
5. Keep Pinyin mode state inside the Pinyin mode/controller.
6. Render composition preview while the Pinyin buffer is active.
7. Render candidate row while candidates are available.
8. Capture letter keys while Chinese Pinyin mode is active.
9. Append captured letters to the Pinyin buffer.
10. Update candidates when the buffer changes.
11. Commit the first candidate on `Space`.
12. Select candidates with number keys.
13. Edit the active buffer with `Backspace`.
14. Cancel the active buffer with `Escape`.
15. Commit unknown Pinyin as plain text when requested.
16. Cancel uncommitted Pinyin state when switching input modes.
17. Skip all Pinyin handling while native system composition is active.
18. Verify `nihao` + `Space` inserts `你好`.
19. Verify number selection works.
20. Verify `Escape` cancels without changing committed text.
21. Verify unknown input can be committed as plain text.
22. Verify system input mode remains unaffected.
23. Run `npm run build`.
